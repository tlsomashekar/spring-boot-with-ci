Pre-Flight Assessment

Run before starting any migration — know what you're dealing with
⚠️
Do this first on every service. Skipping assessment is the #1 cause of migration failures. Each service has a different profile of risk.

📋 Pre-Migration Checklist

  • Git state clean — commit or stash all WIP before starting
    Run: git status
    Critical
  • Full test suite passes on current version
    Run: mvn clean test — note current pass/fail count
    Critical
  • Branch strategy — create dedicated migration branch
    Run: git checkout -b migration/spring3-java21
    Critical
  • Java 17 available on dev machines & CI
    Install: sdk install java 17.0.9-tem or equivalent
    Critical
  • Count javax.* imports to estimate effort
    Run scan script below — <50 = easy, >500 = plan extra days
    Critical
  • Spring Security config style — check for WebSecurityConfigurerAdapter
    Removed in Boot 3 — needs full rewrite
    Critical
  • Springfox/Swagger 2 dependency check
    Must replace with Springdoc — grep -i springfox pom.xml
    Medium
  • Spring Cloud in use? — verify compatibility matrix
    Boot 3.2 → Spring Cloud 2023.x. Check spring.io/projects/spring-cloud
    Medium
  • JUnit 4 tests — Boot 3 ships JUnit 5 by default
    Run: grep -r "import org.junit.Test;" src/test
    Medium
  • Flyway/Liquibase version — Flyway 9.x required for Boot 3
    Boot 3 manages Flyway version — remove explicit version if managed
    Medium
  • RestTemplate usage — consider migrating to WebClient
    Not removed but deprecated. Fine to keep short-term.
    Low
  • H2 in-memory DB for tests — dialect must now be explicit
    Add: spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
    Low
  • Circular bean dependencies — disallowed by default in Boot 2.6+
    Run: grep -r "@Lazy" src/ --include="*.java" to find workarounds
    Low
  • Lombok compatibility — use 1.18.26+ for Java 17/21
    Check: grep -A1 "lombok" pom.xml
    Low

Automated Scan Script

Run on any service — generates a migration risk report
💡
Save as scan-migration.sh, run from project root. Output tells you exactly where to focus effort.
bash · scan-migration.sh 
#!/bin/bash
# ═══════════════════════════════════════════════════
#  Spring Boot / Java Migration — Pre-flight Scanner
#  Usage: chmod +x scan-migration.sh && ./scan-migration.sh
# ═══════════════════════════════════════════════════

set -e
RED='\033[0;31m'; YELLOW='\033[1;33m'; GREEN='\033[0;32m'
BLUE='\033[0;34m'; CYAN='\033[0;36m'; NC='\033[0m'; BOLD='\033[1m'

echo -e "${BOLD}${CYAN}═══════════════════════════════════════════${NC}"
echo -e "${BOLD}  Spring Migration Pre-flight Scanner${NC}"
echo -e "${BOLD}${CYAN}═══════════════════════════════════════════${NC}"

# ── 1. Environment ──────────────────────────────
echo -e "\n${BOLD}[1] Environment${NC}"
echo "  Java:  $(java -version 2>&1 | head -1)"
echo "  Maven: $(mvn -version 2>&1 | head -1)"
echo "  Dir:   $(pwd)"

# ── 2. Current Versions ─────────────────────────
echo -e "\n${BOLD}[2] Project Versions${NC}"
BOOT_VER=$(grep -oP '(?<=)[^<]+' pom.xml | head -3 | tail -1 2>/dev/null || echo "N/A")
echo "  Spring Boot: $(grep -A1 'spring-boot-starter-parent' pom.xml | grep version | grep -oP '(?<=)[^<]+' || echo "N/A")"
echo "  Java target: $(grep '' pom.xml | grep -oP '(?<=)[^<]+' || echo "N/A")"

# ── 3. javax imports (THE BIG ONE) ──────────────
echo -e "\n${BOLD}[3] javax.* Imports${NC}"
JAVAX_COUNT=$(grep -r "import javax\." src/main --include="*.java" 2>/dev/null | wc -l)
if [ "$JAVAX_COUNT" -gt 100 ]; then
  echo -e "  ${RED}HIGH:${NC} $JAVAX_COUNT javax imports — significant effort required"
elif [ "$JAVAX_COUNT" -gt 20 ]; then
  echo -e "  ${YELLOW}MED:${NC} $JAVAX_COUNT javax imports — moderate effort"
else
  echo -e "  ${GREEN}LOW:${NC} $JAVAX_COUNT javax imports — minimal effort"
fi
echo "  Top packages:"
grep -rh "import javax\." src/main --include="*.java" 2>/dev/null \
  | sed 's/import //' | sed 's/\..*//' | sort | uniq -c | sort -rn | head -5 \
  | awk '{print "    "$0}'

# ── 4. Spring Security ──────────────────────────
echo -e "\n${BOLD}[4] Spring Security${NC}"
WSCA=$(grep -rl "WebSecurityConfigurerAdapter" src/ --include="*.java" 2>/dev/null | wc -l)
ANTML=$(grep -rl "antMatchers\|mvcMatchers\|authorizeRequests" src/ --include="*.java" 2>/dev/null | wc -l)
[ "$WSCA" -gt 0 ] && echo -e "  ${RED}CRITICAL:${NC} WebSecurityConfigurerAdapter in $WSCA file(s) — must rewrite" \
  || echo -e "  ${GREEN}OK:${NC} No WebSecurityConfigurerAdapter"
[ "$ANTML" -gt 0 ] && echo -e "  ${YELLOW}WARN:${NC} antMatchers/authorizeRequests in $ANTML file(s) — must update" \
  || echo -e "  ${GREEN}OK:${NC} No deprecated matchers"

# ── 5. Swagger/Springfox ────────────────────────
echo -e "\n${BOLD}[5] Swagger / API Docs${NC}"
SF=$(grep -i "springfox\|swagger-spring" pom.xml 2>/dev/null | wc -l)
SD=$(grep -i "springdoc" pom.xml 2>/dev/null | wc -l)
[ "$SF" -gt 0 ] && echo -e "  ${RED}ACTION:${NC} Springfox found — replace with Springdoc" \
  || echo -e "  ${GREEN}OK:${NC} No Springfox"
[ "$SD" -gt 0 ] && echo -e "  ${GREEN}OK:${NC} Springdoc already present"

# ── 6. Spring Cloud ─────────────────────────────
echo -e "\n${BOLD}[6] Spring Cloud${NC}"
SC=$(grep "spring-cloud" pom.xml 2>/dev/null | wc -l)
[ "$SC" -gt 0 ] && echo -e "  ${YELLOW}WARN:${NC} Spring Cloud found — verify compatibility matrix at spring.io" \
  || echo -e "  ${GREEN}OK:${NC} No Spring Cloud"

# ── 7. JUnit 4 ──────────────────────────────────
echo -e "\n${BOLD}[7] Test Framework${NC}"
J4=$(grep -r "import org.junit.Test;" src/test --include="*.java" 2>/dev/null | wc -l)
[ "$J4" -gt 0 ] && echo -e "  ${YELLOW}WARN:${NC} $J4 JUnit 4 test(s) — migrate to JUnit 5" \
  || echo -e "  ${GREEN}OK:${NC} No JUnit 4 tests detected"

# ── 8. Deprecated Properties ────────────────────
echo -e "\n${BOLD}[8] Deprecated Properties${NC}"
PROPS_FILES=$(find src/main/resources -name "*.properties" -o -name "*.yml" 2>/dev/null)
DEPR_PROPS=("spring.datasource.initialization-mode" "spring.redis." "management.server.base-path" "spring.jpa.mapping-resources")
for f in $PROPS_FILES; do
  for p in "${DEPR_PROPS[@]}"; do
    COUNT=$(grep -c "$p" "$f" 2>/dev/null || true)
    [ "$COUNT" -gt 0 ] && echo -e "  ${YELLOW}WARN:${NC} '$p' in $f — needs rename"
  done
done
echo -e "  (scan complete)"

# ── 9. Flyway / Liquibase ───────────────────────
echo -e "\n${BOLD}[9] Database Migrations${NC}"
grep -i "flyway" pom.xml 2>/dev/null && echo -e "  ${BLUE}INFO:${NC} Flyway detected" || true
grep -i "liquibase" pom.xml 2>/dev/null && echo -e "  ${BLUE}INFO:${NC} Liquibase detected" || true

# ── 10. RestTemplate ────────────────────────────
echo -e "\n${BOLD}[10] HTTP Clients${NC}"
RT=$(grep -rl "RestTemplate" src/main --include="*.java" 2>/dev/null | wc -l)
WC=$(grep -rl "WebClient" src/main --include="*.java" 2>/dev/null | wc -l)
echo "  RestTemplate: $RT file(s) | WebClient: $WC file(s)"
[ "$RT" -gt 5 ] && echo -e "  ${YELLOW}INFO:${NC} Consider migrating RestTemplate → WebClient"

# ── 11. Circular Dependencies ───────────────────
echo -e "\n${BOLD}[11] Circular Dependencies${NC}"
LAZY=$(grep -r "@Lazy" src/ --include="*.java" 2>/dev/null | wc -l)
[ "$LAZY" -gt 0 ] && echo -e "  ${YELLOW}WARN:${NC} $LAZY @Lazy usages — may indicate circular deps (disallowed in Boot 2.6+)" \
  || echo -e "  ${GREEN}OK:${NC} No @Lazy detected"

# ── Summary ─────────────────────────────────────
echo -e "\n${BOLD}${CYAN}═══════════════════════════════════════════${NC}"
echo -e "${BOLD}  Scan Complete — Review items above${NC}"
echo -e "${BOLD}${CYAN}═══════════════════════════════════════════${NC}\n"

Manual Fix — Spring Security

WebSecurityConfigurerAdapter removed — must rewrite as @Bean
🚨
OpenRewrite cannot fully rewrite complex security configs. This is the most common source of compilation errors after running the Boot 3 recipe.
✗ OLD (Boot 2.x)
@Configuration
public class SecurityConfig
  extends WebSecurityConfigurerAdapter {

  @Override
  protected void configure(
      HttpSecurity http) throws Exception {
    http
      .authorizeRequests()
      .antMatchers("/public/**").permitAll()
      .antMatchers("/api/**").authenticated()
      .and()
      .formLogin();
  }
}
✓ NEW (Boot 3.x)
@Configuration
@EnableWebSecurity
public class SecurityConfig {

  @Bean
  public SecurityFilterChain filterChain(
      HttpSecurity http) throws Exception {
    http
      .authorizeHttpRequests(auth -> auth
        .requestMatchers("/public/**").permitAll()
        .requestMatchers("/api/**").authenticated()
        .anyRequest().authenticated()
      )
      .formLogin(withDefaults());
    return http.build();
  }
}

Password Encoder as Bean

java
// OLD: override in WebSecurityConfigurerAdapter
// NEW: expose as @Bean anywhere in @Configuration
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

@Bean
public UserDetailsService userDetailsService() {
    return username -> userRepository
        .findByUsername(username)
        .orElseThrow(() -> new UsernameNotFoundException(username));
}

Java 21 Idioms — Modernise Your Code

Optional but recommended — improves readability and performance
Records — Replace DTOs
// BEFORE: 30-line POJO
public class UserDto {
  private String name;
  private String email;
  // getters, setters, equals...
}

// AFTER: 1-line record
public record UserDto(
    String name,
    String email
) {}
Pattern Matching
// BEFORE
if (obj instanceof String) {
  String s = (String) obj;
  return s.toUpperCase();
}

// AFTER (Java 16+)
if (obj instanceof String s) {
  return s.toUpperCase();
}
Switch Expressions
// BEFORE
String label;
switch (status) {
  case "A": label = "Active"; break;
  default: label = "Other";
}

// AFTER
String label = switch (status) {
  case "A" -> "Active";
  default -> "Other";
};
Text Blocks
// BEFORE: messy SQL/JSON strings
String sql = "SELECT u.id, u.name "
  + "FROM users u "
  + "WHERE u.active = true";

// AFTER: text block
String sql = """
  SELECT u.id, u.name
  FROM users u
  WHERE u.active = true
  """;

Post-Migration Validation

Run after all phases — confirm everything works end-to-end
bash · post-migration-validate.sh
#!/bin/bash
# Post-Migration Validation Script
set -e
GREEN='\033[0;32m'; RED='\033[0;31m'; YELLOW='\033[1;33m'; NC='\033[0m'; BOLD='\033[1m'

pass() { echo -e "${GREEN}✓${NC} $1"; }
fail() { echo -e "${RED}✗${NC} $1"; }
warn() { echo -e "${YELLOW}⚠${NC} $1"; }

echo -e "${BOLD}Post-Migration Validation${NC}\n"

# 1. No javax imports remain
JAVAX=$(grep -r "import javax\." src/main --include="*.java" 2>/dev/null | wc -l)
[ "$JAVAX" -eq 0 ] && pass "No javax.* imports" || fail "$JAVAX javax imports still present"

# 2. No WebSecurityConfigurerAdapter
WSCA=$(grep -r "WebSecurityConfigurerAdapter" src/ --include="*.java" 2>/dev/null | wc -l)
[ "$WSCA" -eq 0 ] && pass "No WebSecurityConfigurerAdapter" || fail "WebSecurityConfigurerAdapter found in $WSCA files"

# 3. No Springfox
SF=$(grep -i "springfox" pom.xml 2>/dev/null | wc -l)
[ "$SF" -eq 0 ] && pass "No Springfox" || fail "Springfox still in pom.xml"

# 4. Java version
JAVA_VER=$(java -version 2>&1 | grep -oP '(?<=version ")[0-9]+')
[ "$JAVA_VER" -ge 21 ] && pass "Java $JAVA_VER" || warn "Java $JAVA_VER (recommend 21)"

# 5. Spring Boot version in pom
BOOT=$(grep -A1 "spring-boot-starter-parent" pom.xml | grep version | grep -oP '(?<=)[^<]+')
echo "  Spring Boot: $BOOT"
[[ "$BOOT" == 3.* ]] && pass "Spring Boot 3.x" || fail "Still on Boot 2.x"

# 6. Build
echo -e "\n${BOLD}Running full build...${NC}"
mvn clean verify -q && pass "Build + Tests PASSED" || fail "Build FAILED"

# 7. Start app (background, wait, health check)
echo -e "\n${BOLD}Starting application...${NC}"
mvn spring-boot:run &> /tmp/app.log &
APP_PID=$!
sleep 15
STATUS=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/actuator/health 2>/dev/null || echo "000")
kill $APP_PID 2>/dev/null
[ "$STATUS" = "200" ] && pass "Actuator /health returned 200" || warn "Actuator health: HTTP $STATUS (check /tmp/app.log)"

echo -e "\n${BOLD}Validation complete.${NC}"
Verify javax gone

curl /actuator/health
Swagger UI
curl /swagger-ui.html
Virtual threads (if enabled)
Thread.currentThread().isVirtual()

Reusable Migration Script

Full automated migration sequence with git checkpoints per phase
♻️
Reuse on any service: Copy migrate.sh to project root, adjust TARGET_BOOT and TARGET_JAVA if needed, and run.
bash · migrate.sh — Full Migration Orchestrator
#!/bin/bash
# ═══════════════════════════════════════════════════════════
#  Spring Boot + Java Migration Orchestrator
#  Reusable across services — runs all phases with checkpoints
#  Usage: chmod +x migrate.sh && ./migrate.sh
# ═══════════════════════════════════════════════════════════

set -e
BOLD='\033[1m'; CYAN='\033[0;36m'; GREEN='\033[0;32m'
RED='\033[0;31m'; YELLOW='\033[1;33m'; NC='\033[0m'

TARGET_BOOT="3.4.1"
TARGET_JAVA="21"
BRANCH="migration/spring${TARGET_BOOT}-java${TARGET_JAVA}"

log()  { echo -e "${CYAN}[migrate]${NC} $1"; }
ok()   { echo -e "${GREEN}[✓]${NC} $1"; }
err()  { echo -e "${RED}[✗]${NC} $1"; exit 1; }
warn() { echo -e "${YELLOW}[⚠]${NC} $1"; }

phase() {
  echo -e "\n${BOLD}${CYAN}══════════════════════════════════════${NC}"
  echo -e "${BOLD}  Phase $1: $2${NC}"
  echo -e "${BOLD}${CYAN}══════════════════════════════════════${NC}"
}

# ── Guard checks ─────────────────────────────────
[ -f "pom.xml" ] || err "No pom.xml found. Run from project root."
command -v mvn &>/dev/null || err "Maven not found"
command -v git &>/dev/null || err "Git not found"
git diff --quiet || err "Uncommitted changes. Commit or stash first."

# ── Create migration branch ───────────────────────
git checkout -b "$BRANCH" 2>/dev/null || git checkout "$BRANCH"
log "Working on branch: $BRANCH"

# ── Run scan first ───────────────────────────────
phase "0" "Pre-flight Scan"
if [ -f "scan-migration.sh" ]; then
  bash scan-migration.sh
  read -p "Review output above. Continue? (y/N): " CONT
  [[ "$CONT" =~ ^[Yy]$ ]] || exit 0
fi

# ── Phase 1: Boot 2.5 → 2.7 ─────────────────────
phase "1" "Spring Boot 2.5 → 2.7"
log "Setting recipe: UpgradeSpringBoot_2_7"
sed -i 's|.*UpgradeSpringBoot.*|org.openrewrite.java.spring.boot2.UpgradeSpringBoot_2_7|' pom.xml
log "Running OpenRewrite dry-run..."
mvn rewrite:dryRun -q
read -p "Apply changes? (y/N): " APPLY
[[ "$APPLY" =~ ^[Yy]$ ]] && mvn rewrite:run -q && ok "Phase 1 changes applied"
mvn clean verify -q && ok "Build green"
git add -A && git commit -m "chore: Spring Boot 2.5→2.7 via OpenRewrite [phase 1]"

# ── Phase 2: Java 17 ─────────────────────────────
phase "2" "Java 8 → 17"
warn "Ensure Java 17 is installed: sdk install java 17.0.9-tem"
read -p "Java 17 ready? (y/N): " JAVA17
[[ "$JAVA17" =~ ^[Yy]$ ]] || exit 0
sed -i 's|.*|org.openrewrite.java.migrate.UpgradeToJava17|' pom.xml
mvn rewrite:run -q
sed -i 's|[^<]*|17|' pom.xml
mvn clean verify -q && ok "Java 17 build green"
git add -A && git commit -m "chore: Java 8→17 via OpenRewrite [phase 2]"

# ── Phase 3: Boot 2.7 → 3.4 (Big One) ──────────
phase "3" "Spring Boot 2.7 → ${TARGET_BOOT} (jakarta + Security)"
warn "This is the biggest phase. Expect manual fixes after."
sed -i 's|.*|org.openrewrite.java.spring.boot3.UpgradeSpringBoot_3_4|' pom.xml
log "Running OpenRewrite (may take 2-5 min)..."
mvn rewrite:run 2>&1 | tee /tmp/phase3-rewrite.log
log "Attempting build (check for remaining errors)..."
mvn clean compile 2>&1 | tee /tmp/phase3-compile.log || warn "Compilation errors remain — fix manually, then re-run build"
git add -A && git commit -m "chore: Spring Boot 2.7→3.4 + javax→jakarta [phase 3]"
warn "Manual fixes needed: Security config, Springdoc, H2 dialect"
warn "Logs: /tmp/phase3-rewrite.log, /tmp/phase3-compile.log"

# ── Phase 4: Java 21 ─────────────────────────────
phase "4" "Java 17 → 21"
warn "Ensure Java 21 is installed: sdk install java 21.0.1-tem"
read -p "Java 21 ready? (y/N): " JAVA21
[[ "$JAVA21" =~ ^[Yy]$ ]] || exit 0
sed -i 's|.*|org.openrewrite.java.migrate.UpgradeToJava21|' pom.xml
mvn rewrite:run -q
sed -i "s|[^<]*|${TARGET_JAVA}|" pom.xml
mvn clean verify -q && ok "Java 21 build green"
git add -A && git commit -m "chore: Java 17→21 via OpenRewrite [phase 4]"

# ── Done ─────────────────────────────────────────
echo -e "\n${BOLD}${GREEN}═══════════════════════════════════════${NC}"
echo -e "${BOLD}${GREEN}  Migration Complete! 🎉${NC}"
echo -e "${BOLD}${GREEN}═══════════════════════════════════════${NC}"
echo -e "Branch: ${CYAN}$BRANCH${NC}"
echo -e "Next: ${YELLOW}Fix any remaining compile errors${NC}"
echo -e "Then: ${YELLOW}Run post-migration-validate.sh${NC}"
echo -e "Then: ${YELLOW}Create PR to main${NC}\n"
